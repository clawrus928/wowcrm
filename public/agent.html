WOW CRM · Agent 操作手冊

給 AI Agent 讀的機器可用說明 · 澳門業務團隊多產品 Pipeline CRM
怎麼用這一頁：把這個網址交給你的 AI agent（ChatGPT / Claude / 自訂 GPT / n8n…）， 對它說「請先讀這個網址，之後照它操作 WOW CRM」，並把你的 API 金鑰交給它。 第一次使用前，使用者需先完成下方〈一次性設定〉。

0.使用者一次性設定（只做一次）

  • 產生金鑰：openssl rand -base64 32
  • GitHub → repo clawrus928/wowcrm → Settings → Secrets and variables → Actions → 新增 API_KEY
  • 重新部署一次（push 任何 commit 或手動跑 deploy workflow）
  • 驗證：curl -H "X-API-Key: 你的金鑰" https://wowcrm.wowmac.com/api/leads → 回 [] 或 JSON 即成功
⚠️ 這把金鑰是管理員權限（可讀寫所有人的資料）。只交給信任的 agent，不要貼在公開的地方。

2.你能做的事（端點）

操作方法 + 路徑
列出全部GET /api/{entity}
取單筆GET /api/{entity}/{id}
新增一筆POST /api/{entity}
批次新增 優先用POST /api/{entity}/bulk · body {"items":[…]} · ≤500 筆 · 整批原子（全成功或全回滾）
更新PATCH /api/{entity}/{id}（只傳要改的欄位）
刪除DELETE /api/{entity}/{id}
移動商機階段POST /api/deals/{id}/stage · body {"stage":"提案中"}
線索轉客戶POST /api/leads/{id}/convert
操作紀錄查詢GET /api/_audit?entity=&recordId=&limit=

entity 可填：leads customers contacts deals quotes contracts channels suppliers pricings activities

3.規則（務必遵守）

  • owner 用 user ID（見下表），盡量都填；不填會掛在 api 名下，前端用戶就改不動。
  • 列舉值用繁體中文原文（見下表），不要翻譯或改字，否則前端篩選對不上。
  • id / created 不用填，系統自動產生。
  • 關聯欄位（customerId、dealId、channelId…）填真實 id；有上下游時先建上游拿 id，再建下游。
  • 大量資料用 bulk、分批 ≤500；回 400 時把錯誤貼給使用者，不要改成逐筆硬塞。
  • 刪除 / 大量更新前先向使用者確認。

6.範例

A · 批次建客戶

curl -s -X POST https://wowcrm.wowmac.com/api/customers/bulk \
  -H "X-API-Key: <你的金鑰>" -H "Content-Type: application/json" -d '{
  "items": [
    {"name":"新街美食","industry":"餐飲","address":"澳門新馬路","owner":"u4"},
    {"name":"鴻運電器","industry":"零售","address":"澳門高士德","owner":"u5"}
  ]
}'

B · 查進行中商機

curl -s -H "X-API-Key: <你的金鑰>" https://wowcrm.wowmac.com/api/deals
# 然後篩 status == "進行中"

C · 標記商機已成交

curl -s -X PATCH https://wowcrm.wowmac.com/api/deals/d_xxx \
  -H "X-API-Key: <你的金鑰>" -H "Content-Type: application/json" \
  -d '{"status":"已成交"}'

7.使用者可以這樣交代

  • 「把這份 CSV 都建成客戶，負責人 Cyrus（u4）。」
  • 「列出所有進行中的商機，按金額排序。」
  • 「把商機 d_xxx 標成已成交。」
  • 「幫『新街美食』新增一筆商機，行銷服務，需求確認階段。」
  • 「這個月新進的線索整理成表。」
