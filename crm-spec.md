# CRM 專案需求規格

## 概述
建立一個多產品 Pipeline CRM 系統，參考微問家（WeiWenJia）CRM 的結構。
使用 React + Vite 建立，亮色主題，類似 Claude App 風格。

## 技術棧
- React 18 + Vite
- 單頁應用，無需後端（先用 mock data）
- 響應式設計（桌面 + 手機）
- 字體：Noto Sans TC + DM Mono

## 六大模組

### 1. 線索 Leads
- 欄位：姓名、商戶名稱、手機、跟進狀態、線索來源、負責人、協作人、創建時間
- 跟進狀態：未處理 / 初訪 / 跟進中 / 報價 / 已轉客戶 / 無效
- 線索來源：官網 / 轉介紹 / 電話開發 / 展覽 / 社群媒體 / 廣告 / 其他

### 2. 客戶 Customers
- 欄位：商戶名稱、corp_group（集團，可選）、所屬行業、地址、跟進狀態、客戶來源、負責人、協作人
- Company 是主鍵，corp_group 欄位連結同集團公司（如鴻海集團 → 鴻海精密 / 群創光電）
- 行業：科技 / 金融 / 零售 / 製造 / 餐飲 / 醫療 / 教育 / 貿易 / 其他

### 3. 聯系人 Contacts
- 欄位：姓名、關聯客戶、職位、座機、Email、負責人、創建時間
- 屬於特定 Customer（1:N）

### 4. 商機 Deals
- 欄位：商機名稱、關聯客戶、產品線、階段、金額、狀態、負責人、協作人、創建時間
- 狀態：進行中 / 已成交 / 已流失
- 每個 Deal 屬於一條產品線，階段跟隨該產品線的 pipeline

### 5. 合同 Contracts
- 欄位：合同名稱、關聯客戶、關聯商機、金額、狀態、簽約日期、開始/到期日期、負責人
- 狀態：草稿 / 審批中 / 已簽署 / 執行中 / 已完成 / 已終止

### 6. 報價單 Quotes
- 欄位：報價單名稱、關聯客戶、關聯商機、金額、狀態、有效期限、負責人
- 狀態：草稿 / 已發送 / 已接受 / 已拒絕 / 已過期

## 三條產品線（各有獨立 Pipeline 階段）

### 顧問服務 Consulting
- 階段：需求分析 → 提案中 → 執行中 → 驗收中 → 已完成
- 顏色：#2563EB（藍）

### 硬體設備 Hardware
- 階段：報價中 → 採購中 → 出貨中 → 安裝中 → 已交付
- 顏色：#059669（綠）

### 行銷服務 Marketing
- 階段：需求確認 → 策略規劃 → 執行中 → 成效追蹤 → 已結案
- 顏色：#D97706（琥珀）

## Pipeline 看板
- 按產品線切換，Kanban 視圖
- 桌面拖曳 + 手機按鈕移動階段
- 每欄顯示數量和金額小計

## UI 規格

### 導航
- 左側邊欄：CRM 管理分區，包含全部 6 模組 + Pipeline
- 手機版：漢堡選單 + 底部導航列

### 列表頁（每個模組）
- Tabs：全部 / 我負責的 / 我協作的
- 篩選欄：下拉篩選器 + 右側搜尋框
- 表格：密集可掃描的行（參考微問家截圖風格）
- 商戶名稱為橘色可點擊連結
